# Contributing

欢迎协作！请先阅读以下规范。

## 分支命名

- `feature/xxx` — 新功能
- `fix/xxx` — 修复
- `docs/xxx` — 文档
- `ref/xxx` — 新增参考资料或创新角度

## Commit 格式

```
<type>: <简短描述>

<可选详细说明>
```

类型：`feat` `fix` `docs` `ref` `chore`

示例：
```
feat: add Korean language support to README
ref: add 3 innovation angles for 环保 topic
```

## PR 流程

1. 从 `main` 拉分支
2. 改完 push，开 PR 到 `main`
3. 需至少 1 人 approve 后合并
4. PR 标题用 commit 格式

## 贡献方向

- **SKILL.md** — 工作流逻辑、交互规则
- **references/** — 补充结构模板、论证方法、创新角度、反模式案例
- **README.md** — 翻译、润色多语言段落
- **创新角度库** (`innovation-checklist.md`) — 新场景、新角度的非主流切入

## 红线

- SKILL.md 中的工作流逻辑必须有充分理由才可改动
- 创新角度必须"站得住脚"——可论证、不牵强
- 论据示例不得虚构名人名言、数据、案例
