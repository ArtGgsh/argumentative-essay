# 中文议论文结构模板

## 一、应试型：是什么 → 为什么 → 怎么办

适用：高考作文、公考申论、考研政治大题等限时写作场景。

### 结构骨架

| 部分 | 功能 | 篇幅 |
|------|------|------|
| 是什么 | 界定问题/概念，明确讨论范围 | ~15% |
| 为什么 | 分析原因/意义/危害，核心论证区 | ~55% |
| 怎么办 | 提出对策/建议/呼吁 | ~30% |

### 操作要点

- **"是什么"不要写成名词解释**：用现象描述或设问引入，点出问题的现实切口。例如不谈"创新是……"，而从"当一家企业十年只做一种产品，它是专注还是停滞？"切入。
- **"为什么"是得分重点**：至少拆 2–3 个分论点，每个配事实论据 + 道理论据。高考作文侧重"为什么会这样"的因果分析；申论侧重"为什么必须解决"的紧迫性论证。
- **"怎么办"要有层次**：个人层面 → 社会层面 → 制度层面，避免"让我们从现在做起"这类空洞呼吁。申论对策必须可操作。

### 变体：驳论三段式

| 部分 | 功能 |
|------|------|
| 摆靶子 | 准确转述对方观点（不丑化、不简化） |
| 析谬误 | 逐层批驳：事实谬误 → 逻辑谬误 → 价值谬误 |
| 立己论 | 提出替代观点并论证 |

## 二、时评型：现象 → 本质 → 出路

适用：新闻时评、社会评论、公众号评论文章。

### 结构骨架

| 部分 | 功能 |
|------|------|
| 现象 | 用具体个案切入，一两句话勾勒事件 |
| 本质 | 从个案上升到结构性问题：制度缺陷？文化惯性？利益格局？ |
| 出路 | 不止于批评，给出建设性方向 |

### 操作要点

- 现象描述尽量用**白描**，不带情绪形容词——让事实自己说话。
- 本质分析避免"归根到底是体制问题"这种万能答案，限定在可论证的范围。
- 出路不要求完整方案，但要给出一个可行的思考方向。

---

## 三、学术型：引论 → 本论 → 结论

适用：学术论文、学术随笔、深度长文。

### 结构骨架

| 部分 | 功能 |
|------|------|
| 引论 | 提出问题 + 文献/背景简述 + 中心论点 |
| 本论 | 分论点递进展开，每个分论点含：论点句 → 论据 → 分析 → 小结 |
| 结论 | 总结论证 + 指出局限 + 展望 |

### 操作要点

- 引论必须界定核心概念的操作性定义（本文在什么意义上使用这个词）。
- 本论按逻辑递进排列分论点，不做并列罗列。
- 结论指出局限不是自我否定，而是表明论证的边界意识。

---

## 四、自由型：递进 / 并列 / 对照

适用：思想随笔、公众号深度文、无固定格式要求的议论。

### 三种逻辑

| 逻辑 | 说明 | 示例 |
|------|------|------|
| 递进 | 由浅入深、由表及里 | 个人选择 → 社会规范 → 文明逻辑 |
| 并列 | 多维度展开，维度间无强弱顺序 | 经济影响 → 文化影响 → 心理影响 |
| 对照 | 正反对比，在张力中推进 | 东方路径 vs 西方路径 → 各自的前提条件 → 可否融合 |

### 操作要点

- 递进式每段开头要明确"上一层说了什么，这一层进一步说什么"。
- 并列式避免变成清单——每个维度内部仍然要有论证。
- 对照式最终要给出综合判断，不只停留在"各有优劣"。
